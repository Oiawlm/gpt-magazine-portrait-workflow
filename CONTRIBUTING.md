# 贡献指南
欢迎参与 gpt-magazine-portrait-workflow 项目的贡献！无论是提交 Issue、改进文档、贡献新风格包，还是优化代码，我们都非常欢迎。
## 贡献方式
### 1. 提交 Issue
- 遇到问题或有改进建议时，请先搜索现有 Issue 是否已经存在
- 如果没有，创建新 Issue，详细描述问题或建议，包括复现步骤、期望效果等
### 2. 改进文档
- 文档错误、表述不清、缺失内容都可以改进
- 包括 README、WORKFLOW、各类指南和示例
### 3. 贡献风格包
- 可以提交高质量的杂志风格参考图到 `assets/style-reference/` 目录
- 风格图要求：分辨率 ≥1024x1024，风格明确可复用，适合做杂志写真参考
- 提交时请同步更新 `assets/style-reference/STYLE_PACKS.md`，说明风格特点和适用场景
### 4. 优化 Skill 脚本
- 改进现有脚本的功能和稳定性
- 新增辅助脚本，提高工作流效率
- 代码请遵循 PowerShell 最佳实践，添加必要的注释
## 提交规范
### Pull Request 流程
1. Fork 本仓库到自己的 GitHub 账号
2. 创建新的功能分支：`git checkout -b feature/your-feature-name`
3. 进行修改，提交代码：`git commit -m "描述你的修改"`
4. 推送到自己的仓库：`git push origin feature/your-feature-name`
5. 提交 Pull Request 到本仓库的 main 分支
### 提交信息规范
- 使用清晰的中文描述提交内容
- 格式：`[类型] 内容描述`
- 类型包括：
  - `feat`：新增功能
  - `fix`：修复问题
  - `docs`：文档改进
  - `style`：格式调整（不影响代码逻辑）
  - `refactor`：代码重构
  - `asset`：新增/更新资产文件
## 注意事项
1. **不要提交和本工作流无关的素材**：资产应服务于人物杂志写真生成流程
2. **不要提交个人临时试跑垃圾文件**：只提交可复用样张、风格包或文档
3. **不要修改核心流程规范**：如果要修改工作流的核心逻辑，请先提交 Issue 讨论
4. **保持文档同步**：如果修改了流程或功能，请同步更新对应的文档
## 开发环境
- PowerShell 7.0+（用于运行脚本）
- Git（用于版本控制）
- VS Code（推荐编辑器，安装 Markdown 相关插件）
## 许可证
你提交的内容将默认采用 MIT 许可证开源。
