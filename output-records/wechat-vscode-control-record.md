# 微信控制 VS Code 尝试记录

- 日期：2026-06-05
- 记录：尝试研究通过 Cherry Studio 的微信频道控制电脑上的 VS Code / Claude Code 工作流。
- 结论：暂时不继续使用该方案。
- 备注：本次只读检查了 Cherry Studio、Claude CLI、Codex 的模型、Skill、MCP 配置边界。

## 实测补充

- 当前微信频道测试绑定的是 `deepseek-v4-pro` 模型。
- 当前模型与链路不能识图。
- 微信端可以向电脑端发送文字和图片。
- 电脑端 Agent 当前无法理解微信端发来的图片内容。
- 电脑端 Agent 当前也不能把电脑上的图片作为图片消息发送回微信。
- 因此现阶段该方案更适合作为文字遥控入口：用微信发送文字指令、接收文字结果；不适合图片分析、图片审核或图片回传工作流。
