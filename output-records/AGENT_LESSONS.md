# Agent Lessons

本文件记录用户指出的错误、Agent 明确意识到的协作偏差，以及会影响后续工作的默会知识。文件名用英文，正文用中文，方便用户翻阅。

## 记录规则

- 只记录会改变未来行为的错误、偏差和默会知识，不记录普通命令失败。
- 适用于所有 Agent，包括 Codex、Claude Code、Cherry Studio Agent，以及后续加入的其他自动化助手。
- 每条记录尽量包含：日期、Agent、范围、错误、原因、修正、以后规则、同步动作。
- 如果错误影响全局协作规则，应同步更新根目录 `AGENTS.md` 和 `CLAUDE.md`。
- 如果错误只属于某个项目，应同步更新该项目的 `PROJECT_GUIDE.md`。
- 如果只是对话流水、试跑记录、交接说明或决策沉淀，应写入 `output/`，不要混进具体项目代码目录。
- 记录目的不是追责，而是把用户的默会知识变成可复用规则，避免重复犯错。

### 2. 生图前必须先列清单，不要先斩后奏

Agent: Codex  
范围: AI 生图工作流 / 额度节省

错误：

在没有先列出生图数量、角色、风格、输出路径和覆盖规则的情况下，直接执行了额外生图。

原因：

把“主动推进”理解成“直接执行”，没有区分低成本文本整理和高成本生图动作。

修正：

生图属于高成本动作，必须先列清单让用户确认。

以后规则：

每次生图前必须先说明：

- 本次生成几个角色；
- 每个角色生成几个风格；
- 输出到哪个目录；
- 是否覆盖旧图；
- 哪些步骤可以交给 Claude Code / DeepSeek / 豆包做。

用户确认后再执行。

同步动作：

已在 `AGENTS.md` 和 `CLAUDE.md` 写入“生图前必须先列清单并等待用户确认”。

### 4. 不要误判 Claude Code 的模型能力

Agent: Codex  
范围: Claude Code 分工 / 模型能力边界

错误：

一开始把 Claude Code 误判为只接入 DeepSeek V4 Pro，并据此说 Claude Code 只能做纯文本任务。

原因：

没有记住用户说明：Claude Code 可通过 CC Switch 切换模型，当前还接入了 Doubao-Seed-2.0-Pro 和豆包 4.2.0。

修正：

Claude Code 不是固定等于 DeepSeek。它可通过 CC Switch 切换模型。DeepSeek V4 Pro 是纯文本模型，不能收发图片；豆包模型是否能看图，取决于当前 Claude Code 会话和插件是否支持图片输入。

以后规则：

讨论 Claude Code 分工时，必须先区分：

- 当前 Claude Code 切到哪个模型；
- 该模型是否支持图片输入；
- 当前会话/插件是否真的能上传和读取图片；
- 任务是文本整理、看图风格吸收、代码整理，还是最终生图。

同步动作：

已在 `AGENTS.md`、`CLAUDE.md` 和 `output/2026-06-05-人物写真自动生图试跑.md` 修正模型能力说明。

### 5. ChatGPT 桌面版自动化只是降级方案，不应默认作为主路线

Agent: Codex  
范围: AI 生图执行路线

错误：

在已知 Codex 可直接生图的前提下，把“Codex 操作 ChatGPT 桌面版出图”当成第一版主路线来推进。

原因：

过度关注桌面自动化可行性，忽略了这会增加步骤、消耗时间，并绕开 Codex 直接生图能力。

修正：

如果 Codex 当前能直接使用生图能力，应优先让 Codex 直接生图、保存和归档。ChatGPT 桌面版自动化只在直接生图不可用，或用户明确要求使用桌面版时作为降级路线。

以后规则：

设计生图流水线时，优先选择步骤最少、额度最省、失败点最少的路线。不要为了自动化而自动化。

同步动作：

已在 `AGENTS.md`、`CLAUDE.md` 和试跑记录中把 ChatGPT 桌面版路线标记为降级方案。
